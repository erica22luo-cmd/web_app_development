# 讀書筆記本 - 產品需求文件 (PRD)

## 1. 專案概述

### 背景與動機
在學校圖書館借閱書籍時，學生常常希望能記錄自己讀過的書籍、對書籍的評價與心得，並透過這些紀錄幫助其他學生或自己在未來找到合適的書籍。為了解決知識碎裂與挑書困難的問題，我們規劃開發這款「讀書筆記本」系統。

### 目標用戶
- 學校圖書館的學生。
- 尋找推薦書籍的校園讀者。

### 核心價值主張
提供一個簡單易用的集中管理平台，讓學生可以輕鬆紀錄書名、撰寫心得並留下評分。同時，強大的搜尋機制能幫助學生探索他人紀錄，讓每個人都能快速找到自己心儀的書籍。

## 2. 功能需求

以下為系統核心的主要功能需求清單，包含對應的使用者故事 (User Story)：

### 1. 記錄書名與書籍基本資訊
- **作為** 學生，
- **我希望** 能夠在系統中新增我讀過的書籍名稱，
- **以便** 為我的閱讀歷程留下紀錄。

### 2. 撰寫閱讀心得
- **作為** 學生，
- **我希望** 能夠針對我讀過的書撰寫個人的閱讀筆記與心得，
- **以便** 在未來能回顧自己對這本書的想法與收穫。

### 3. 滿意度評分
- **作為** 學生，
- **我希望** 能夠對閱讀完的書籍給予滿意度評分（如1~5星），
- **以便** 用直觀的方式表達對這本書的喜好程度。

### 4. 書籍與心得搜尋
- **作為** 想找書的學生，
- **我希望** 能夠透過關鍵字搜尋（例如：書名等）來尋找書籍與筆記，
- **以便** 快速找到我想看的書或參考他人的見解。

### 5. 書籍分類與標籤
- **作為** 學生，
- **我希望** 能夠為我的讀書筆記加上分類或標籤，
- **以便** 可以更有系統地管理與篩選特定的閱讀紀錄。

## 3. 非功能需求

### 技術限制
- **後端框架**：Flask (Python)
- **前端呈現**：Jinja2 模板與 HTML / CSS / JS
- **資料庫**：SQLite

### 效能與安全考量
- **效能考量**：頁面載入與搜尋結果應於短時間內回傳（如 2 秒以內），保持順暢的體驗。
- **安全考量**：防範基礎的 SQL Injection 及 XSS 攻擊，確保學生撰寫的心得紀錄不會遭到惡意竄改。

## 4. MVP 範圍 (Minimum Viable Product)

| 等級 | 功能項目 |
|------|-----------|
| **Must Have** (核心必備) | 1. 記錄書名<br>2. 撰寫與顯示閱讀心得<br>3. 書籍關鍵字搜尋 |
| **Should Have** (重要需有) | 1. 滿意度評分功能<br>2. 基礎的書籍分類機制 |
| **Nice to Have** (有會更好) | 1. 好書熱門排行榜<br>2. 書摘清單匯出功能 |
